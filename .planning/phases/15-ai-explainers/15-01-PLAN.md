  <action>
1. Create migration `0027_ai_explainers.sql`:
   - Table `standard_explanations`: id TEXT PK, standard_id TEXT NOT NULL, accreditor TEXT, plain_english TEXT NOT NULL, required_evidence JSON NOT NULL, common_mistakes JSON, regulatory_context TEXT, confidence REAL DEFAULT 0.85, version TEXT (standard body hash for cache invalidation), created_at TEXT, updated_at TEXT
   - Index on (standard_id, version) for cache lookups
   - FOREIGN KEY to standards(id) with ON DELETE CASCADE

2. Create `src/services/standard_explainer_service.py`:
   - Class `StandardExplainerService` with `__init__(ai_client, standards_store)`
   - Method `explain_standard(standard_id: str) -> Dict[str, Any]`:
     - Check cache first (by standard_id + version hash)
     - If miss: call AIClient.generate() with system prompt from research (JSON output format)
     - Parse JSON response, extract plain_english, required_evidence list, common_mistakes, regulatory_context
     - Store in cache table with version = hashlib.sha256(standard.body).hexdigest()[:16]
     - Return dict with all fields + confidence
   - Method `get_cached_explanation(standard_id: str) -> Optional[Dict]` for direct cache lookup
   - Method `invalidate_cache(standard_id: str)` for manual refresh
   - Use existing pattern from readiness_service.py (dataclass for response, to_dict())

3. Create test file `tests/test_standard_explainer_service.py`:
   - Mock AIClient.generate() to return valid JSON
   - Test explain_standard returns required fields (plain_english, required_evidence, common_mistakes)
   - Test caching: second call doesn't invoke AI
   - Test cache invalidation: subsequent call invokes AI again
  </action>
