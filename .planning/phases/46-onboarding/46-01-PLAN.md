# Plan 46-01: Onboarding Wizard

## Objective

Create 4-step onboarding wizard for new institutions: Profile → Upload → Audit → Review.

## Dependencies

- Phase 41 (Auth) for user context
- Phase 42 (RBAC) for permission checks

## Tasks

### Task 1: Database Migration

**File:** `src/db/migrations/0051_onboarding.sql`

```sql
CREATE TABLE IF NOT EXISTS onboarding_progress (
    id TEXT PRIMARY KEY,
    institution_id TEXT NOT NULL UNIQUE,
    current_step INTEGER NOT NULL DEFAULT 1,
    completed INTEGER NOT NULL DEFAULT 0,
    profile_complete INTEGER NOT NULL DEFAULT 0,
    documents_uploaded INTEGER NOT NULL DEFAULT 0,
    initial_audit_run INTEGER NOT NULL DEFAULT 0,
    review_complete INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (institution_id) REFERENCES institutions(id)
);
```

**Commit:** `chore(46): add onboarding_progress table`

### Task 2: Onboarding Service

**File:** `src/services/onboarding_service.py`

Implement:
- `start_onboarding(institution_id)` → create progress record
- `update_step(institution_id, step, data=None)` → update progress
- `get_progress(institution_id)` → current progress
- `is_onboarding_complete(institution_id)` → bool

### Task 3: Onboarding API Blueprint

**File:** `src/api/onboarding.py`

Blueprint: `onboarding_bp`, prefix `/api/onboarding`

Endpoints:
- `GET /progress` — current progress

**Commit:** `feat(46): add onboarding API blueprint`

### Task 4: Onboarding Wizard UI

**File:** `templates/onboarding.html`

Full-page wizard with step indicator (1 → 2 → 3 → 4).

**Step 1: Institution Profile**
- Institution name
- Primary accreditor (dropdown: ACCSC, SACSCOC, HLC, ABHES, COE)
- State/territory
- Programs (add multiple: name + credential level)
- Next visit date (approximate)
- Save → creates institution + workspace

**Step 2: Upload Key Documents**
- Guided upload for:
  - Enrollment Agreement(s)
  - Institutional Catalog
  - Student/School Handbook
  - Faculty Credentials (optional)
- Drag-drop area
- Auto-classify uploaded documents
- Progress with type detection badges
- "Skip for now" option (warning)

**Step 3: Initial Audit**
- "Run your first audit" button
- SSE-streamed progress (reuse existing UI)
- Results inline: severity counts, top 5 critical
- This is the "wow moment"

**Step 4: Review & Next Steps**
- Audit summary
- Prioritized next actions (from `get_next_actions`)
- "Invite your team" CTA
- "Set up compliance calendar" CTA
- "Explore the dashboard" → redirect to `/`

Design: Certified Authority theme (dark, gold accent, Cormorant Garamond).

**Commit:** `feat(46): add onboarding wizard UI`

### Task 5: App Integration

**File:** `app.py`

Add redirect logic:
```python
@app.before_request
def check_onboarding():
    if not app.config.get('AUTH_ENABLED', True):
        return None
    # Skip exempt paths
    if request.path.startswith(('/api/auth', '/api/onboarding', '/static', '/login', '/register', '/onboarding')):
        return None
    user = g.get('current_user')
    if user and onboarding_service.should_show_onboarding(user['id']):
        return redirect('/onboarding')
```
