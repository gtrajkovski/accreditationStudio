## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Scope badge padding, inline icon gaps |
| sm | 8px | Tab spacing, filter chip gaps |
| md | 16px | Search input padding, result item padding |
| lg | 24px | Results panel section padding |
| xl | 32px | Command palette internal margins |

Exceptions: **none** — Phase 27 uses only existing spacing tokens from variables.css.

**Touch targets:** 44px minimum for interactive elements (Tab key cycle trigger area, close button, result items). Achieved via sm padding (8px) + md font (16px) + vertical padding (16px) = 48px total.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (results) | 16px (--font-size-base) | 400 | 1.5 |
| Label (scope badge) | 14px (--font-size-sm) | 600 | 1.2 |
| Placeholder | 16px (--font-size-base) | 400 | 1.5 |

**Rationale:** Results must be readable at a glance. Scope badge is uppercase + semibold for prominence. Font sizes declared: 12px, 14px, 16px (3 sizes). Weights declared: 400, 600 (2 weights). Matches design contract limits.

## Component Inventory

### 1. Scope Badge Component
**Location:** New file `static/js/scope_badge.js`

**Visual contract:**
- Pill-shaped (border-radius: 9999px)
- Border: 1px solid var(--accent-primary)
- Background: transparent (dark mode), var(--accent-muted) (light mode)
- Padding: 4px 8px (xs horizontal, sm vertical)
- Font: 14px semibold, uppercase, letter-spacing: 0.02em
- Icon: Chevron-down SVG (8px × 8px) to right of text
- Hover state: Background: var(--accent-muted)
- Active state (during Tab press): Background: var(--accent-primary), text: var(--bg-base)

**Behavior:**
- Renders current scope from 6-level enum (global, institution, program, document, standards, compliance)
- Cycles to next scope when Tab key pressed (while search input focused)
- Emits `scope-changed` custom event with new scope value
- Updates placeholder text in sibling search input

**Accessibility:**
- `aria-label="Search scope: {current scope}"`
- `aria-live="polite"` on scope label (announces changes to screen readers)
- `aria-keyshortcuts="Tab"` on container

### 2. Inline Search Bar Component
**Location:** New file `static/js/contextual_search.js`

**Visual contract:**
- Width: 100% of header (max-width: 600px)
- Height: 44px (meets touch target minimum)
- Background: var(--bg-input)
- Border: 1px solid var(--border-subtle), focus: 2px solid var(--accent-primary)
- Border-radius: 8px (--radius-md)
- Search icon: 20px × 20px, positioned left (16px from edge)
- Scope badge: positioned right inside input (8px from edge)
- Padding: 12px 120px 12px 48px (left space for icon, right space for badge)
- Placeholder: "Search {scope}..." (e.g., "Search This Program...")
- Font: 16px regular, color: var(--text-primary)

**States:**
- Default: Border subtle, placeholder visible
- Focus: Border accent (2px), placeholder remains
- Typing: Placeholder hidden, debounced search starts after 250ms
- Loading: Subtle spinner in badge position
- Results: Results panel slides down below input (with 8px gap)
- Error: Border red (var(--error)), error message below input

**Keyboard behavior:**
- `/` key (global): Focus input
- `Ctrl+K` (global): Open command palette (higher precedence than inline search)
- `Tab` (when input focused): Cycle scope, prevent default tab navigation
- `Shift+Tab`: Standard reverse tab navigation (do not intercept)
- `Escape`: Clear input, close results panel
- `Shift+Up/Down`: Navigate result items (when results visible)
- `Enter`: Trigger selected result action

**Debouncing:**
- 250ms delay after last keystroke
- Cancel in-flight requests when new query starts (race condition prevention via sequential searchId)

## Interaction Patterns

### Pattern 1: Scope Cycling with Tab Key
**Trigger:** User presses Tab while search input has focus (without Shift modifier)

**Flow:**
1. Prevent default Tab navigation (`e.preventDefault()`)
2. Increment scope index: `(currentIndex + 1) % 6`
3. Update badge label via i18n: `t('search.scope.' + newScope)`
4. Update placeholder: `t('search.placeholder.' + newScope)`
5. Emit `scope-changed` event
6. If query.length >= 2: Re-execute search with new scope
7. Announce to screen reader: "Scope changed to {newScope}"

**Edge case:** If user holds Tab (repeated keydown), debounce cycle to max 1 per 300ms to prevent accidental multi-cycle.

### Pattern 2: Debounced Search with Race Prevention
**Trigger:** User types in search input

**Flow:**
1. Clear existing timeout: `clearTimeout(searchTimeout)`
2. If query.length < 2: Show min length hint, return
3. Set new timeout (250ms): `searchTimeout = setTimeout(() => executeSearch(), 250)`
4. In executeSearch: Increment `currentSearchId`, store as `searchId`
5. Show loading state (spinner or skeleton)
6. POST `/api/search/contextual` with query, scope, context IDs
7. On response: Check if `searchId === currentSearchId` (abort if stale)
8. Render results with tabs and counts
9. Focus first result item

**Error handling:**
- Network error: Show error state with retry button
- 400 (validation): Show error message from API
- 500: Generic "Search failed" message

### Pattern 3: Keyboard Navigation in Results
**Trigger:** User presses Shift+Up or Shift+Down while results visible

**Flow:**
1. Prevent default scroll behavior (`e.preventDefault()`)
2. Calculate new index: `selectedIndex + (key === 'Down' ? 1 : -1)`
3. Clamp index: `Math.max(0, Math.min(selectedIndex, results.length - 1))`
4. Remove `.selected` class from previous item
5. Add `.selected` class to new item
6. Scroll selected item into view: `element.scrollIntoView({ block: 'nearest' })`
7. Update `aria-activedescendant` on results container

**Edge case:** If results panel not visible, Shift+Up/Down do nothing (no-op).

### Pattern 4: Focus Trap in Command Palette
**Trigger:** User opens command palette (Ctrl+K)

**Flow:**
1. Add `.open` class to palette backdrop
2. Store previously focused element: `previouslyFocused = document.activeElement`
3. Focus search input
4. Add `keydown` listener to palette container
5. On Tab (forward): If last focusable element, wrap to first
6. On Shift+Tab (backward): If first focusable element, wrap to last
7. On Escape: Remove focus trap, restore focus to `previouslyFocused`, close palette

**Focusable elements:** Search input, tab buttons, result items, close button.
