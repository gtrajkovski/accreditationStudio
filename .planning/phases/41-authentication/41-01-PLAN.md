## Tasks

### Task 1: Database Migration

**File:** `src/db/migrations/0046_users.sql`

```sql
-- User authentication tables

CREATE TABLE IF NOT EXISTS users (
    id TEXT PRIMARY KEY,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    name TEXT NOT NULL,
    role TEXT NOT NULL DEFAULT 'viewer',
    institution_id TEXT,
    is_active INTEGER NOT NULL DEFAULT 1,
    last_login TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (institution_id) REFERENCES institutions(id)
);

CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    token TEXT UNIQUE NOT NULL,
    expires_at TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS password_resets (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    token TEXT UNIQUE NOT NULL,
    expires_at TEXT NOT NULL,
    used INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_users_email ON users(email);
CREATE INDEX IF NOT EXISTS idx_users_institution ON users(institution_id);
CREATE INDEX IF NOT EXISTS idx_sessions_token ON sessions(token);
CREATE INDEX IF NOT EXISTS idx_sessions_user ON sessions(user_id);
CREATE INDEX IF NOT EXISTS idx_password_resets_token ON password_resets(token);
```

**Commit:** `chore(41): add users, sessions, password_resets tables`

### Task 2: Auth Service

**File:** `src/services/auth_service.py`

Implement:
- `register_user(email, password, name, institution_id=None)` → hash with werkzeug.security, create user
- `authenticate(email, password)` → verify, create session token (UUID4), 24hr expiry
- `validate_session(token)` → check sessions table, verify not expired
- `logout(token)` → delete session
- `request_password_reset(email)` → generate reset token, 1hr expiry
- `reset_password(token, new_password)` → validate, update hash, invalidate sessions
- `get_user(user_id)` → return user without password_hash
- `update_user(user_id, updates)` → update name, email

Password: minimum 8 characters. Use `werkzeug.security.generate_password_hash` / `check_password_hash`.

**Commit:** `feat(41): implement auth service`

### Task 3: Auth API Blueprint

**File:** `src/api/auth.py`

Blueprint: `auth_bp`, prefix `/api/auth`

Endpoints:
- `POST /register` — `{email, password, name}` → `{user, token}`
- `POST /login` — `{email, password}` → `{user, token}`
- `POST /logout` — requires auth → `{success: true}`
- `GET /me` — requires auth → current user
- `POST /forgot-password` — `{email}` → `{success: true}` (always 200)
- `POST /reset-password` — `{token, password}` → `{success: true}`

Auth header: `Authorization: Bearer <token>`

DI pattern: `init_auth_bp(auth_service)`

**Commit:** `feat(41): add auth API blueprint`

### Task 5: App Integration

**File:** `app.py`

1. Import and register `auth_bp`
2. Add `AUTH_ENABLED` config (default `True`)
3. Create `login_required` decorator:
   - Check `Authorization` header for API
   - Check session cookie for pages
   - Return 401 / redirect to `/login`
   - Exempt: `/login`, `/register`, `/api/auth/*`, `/api/health`, static
4. First-run check: if no users, redirect to `/register`
5. Inject `current_user` into Jinja2 context

**Critical:** When `AUTH_ENABLED=False`, skip all auth (preserves dev workflow).

**Commit:** `feat(41): integrate auth into app`
