# Plan 21-01: validate_packet() + Export Gating

## Goal
Implement packet validation that blocks export unless all evidence requirements are met, with explicit override checkpoint.

## Requirements Covered
- EVID-01: Packet export blocked unless all selected standards have evidence
- EVID-02: Critical findings must be resolved or explicitly waived before export

## Tasks

### 1. validate_packet() Function
- **File:** `src/services/packet_service.py` (new or extend existing)
- **Function:** `validate_packet(packet_id: str) -> ValidationResult`
- **Returns:**
  ```python
  @dataclass
  class ValidationResult:
      ok: bool
      missing_standards: List[str]  # Standards without evidence
      missing_evidence: List[str]   # Findings without evidence refs
      blocking_findings: List[str]  # Critical/unresolved findings
      required_checkpoints: List[str]  # Checkpoints that must be resolved
  ```

### 2. Validation Logic
- Query packet's selected standards
- For each standard: check `evidence_refs` table for at least one mapping
- Query `audit_findings` for critical findings with status != resolved/dismissed
- If any blocking issues exist: `ok = False`

### 3. Export Endpoint Gating
- **File:** `src/api/packets.py`
- **Action:** Modify export endpoints to call `validate_packet()` first
- **Response:** If `ok=False`, return 400 with validation errors
- **Override:** Accept `?force=true&checkpoint_id=XXX` with valid `finalize_submission` checkpoint

### 4. finalize_submission Checkpoint Type
- **File:** `src/core/models.py`
- **Action:** Add checkpoint type constant
- **Checkpoint creation:** When user clicks "Export Anyway", create checkpoint for audit trail

### 5. Unit Tests
- **File:** `tests/test_packet_validation.py`
- **Cases:**
  - Packet with full evidence → ok=True
  - Packet missing evidence → ok=False with list
  - Packet with critical findings → ok=False
  - Force export with checkpoint → ok (logged)

- [ ] Audit trail records forced exports
