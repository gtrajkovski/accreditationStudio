# Plan 21-02: Coverage Step UI in Packet Studio

## Goal
Add a visual coverage verification step in Packet Studio that shows standards with/without evidence and gates the export button.

## Requirements Covered
- EVID-03: Coverage step in Packet Studio shows gaps visually

## Tasks

### 1. Coverage Step Component
- **File:** `templates/partials/packet_coverage.html`
- **Layout:**
  - Standards list with evidence count per standard
  - Progress bar: "X of Y standards covered"
  - Missing items highlighted in red/yellow
  - Collapsible detail sections

### 2. Standards Coverage List
- **Display:**
  - Standard code + title
  - Evidence count badge (green if >= 1, red if 0)
  - Expand to show evidence document links
  - "Add Evidence" button for gaps

### 3. Blocking Issues Panel
- **Display:**
  - Critical findings (if any)
  - Required checkpoints
  - "Resolve" links

### 4. Export Button State
- **File:** `static/js/packet_studio.js`
- **Logic:**
  - Call `/api/packets/<id>/validate` on step enter
  - If `ok=false`: disable Export, show "Resolve Issues" message
  - If `ok=true`: enable Export button

### 5. Force Export Modal
- **Trigger:** "Export Anyway" link (visible only if issues exist)
- **Content:**
  - List of issues being waived
  - Confirmation checkbox
  - "I understand" button → creates checkpoint, enables export

### 6. CSS Styling
- **File:** `static/css/components/packet_coverage.css`
- **Styles:** Coverage bars, status badges, issue highlights

## Verification
- [ ] Coverage step shows all standards with evidence counts
- [ ] Missing evidence items highlighted
- [ ] Export button disabled when validation fails
- [ ] Force export modal creates checkpoint
- [ ] Mobile-responsive layout
