<task type="auto">
  <name>Task 2: Create global search API blueprint</name>
  <files>src/api/global_search.py</files>
  <action>
Create global_search_bp Blueprint with these endpoints:

1. POST `/api/institutions/<institution_id>/global-search`
   - Body: {query, filters: {sources, doc_types, compliance_status, date_range}, limit, offset}
   - Calls SiteVisitService.search() with enhanced filters
   - Groups results by source_type with counts
   - Returns: {results: [], total, query_time_ms, sources_searched, grouped_counts: {documents: N, standards: N, ...}}

2. GET `/api/institutions/<institution_id>/global-search/recent`
   - Returns last 5 searches from site_visit_searches table
   - For display in empty command palette state

3. GET `/api/institutions/<institution_id>/global-search/presets`
   - Returns all filter presets for institution
   - Order by usage_count DESC, name ASC

4. POST `/api/institutions/<institution_id>/global-search/presets`
   - Body: {name, filters}
   - Creates or updates preset (ON CONFLICT)
   - Returns created/updated preset

5. DELETE `/api/institutions/<institution_id>/global-search/presets/<preset_id>`
   - Deletes preset
   - Returns 204 No Content

6. POST `/api/institutions/<institution_id>/global-search/presets/<preset_id>/use`
   - Increments usage_count, updates last_used_at
   - Returns updated preset

Follow existing blueprint pattern (init_global_search_bp dependency injection).
Use generate_id("sfp") for filter preset IDs.
Import get_site_visit_service from site_visit_service.
