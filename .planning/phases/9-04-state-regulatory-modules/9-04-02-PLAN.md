**Authorization Endpoints:**

1. `GET /api/state-regulations` - List all state authorizations for an institution
   - Query param: institution_id (required)
   - Returns: { authorizations: [...], count: N }

2. `POST /api/state-regulations` - Add a state authorization
   - Body: { institution_id, state_code, authorization_status, sara_member, effective_date?, renewal_date?, contact_agency?, contact_url?, notes? }
   - Returns: { authorization: {...} }, 201

3. `GET /api/state-regulations/<state_code>` - Get authorization details
   - Query param: institution_id (required)
   - Returns: { authorization: {...} } or 404

4. `PUT /api/state-regulations/<state_code>` - Update authorization
   - Query param: institution_id (required)
   - Body: { authorization_status?, sara_member?, effective_date?, renewal_date?, contact_agency?, contact_url?, notes? }
   - Returns: { authorization: {...} }

5. `DELETE /api/state-regulations/<state_code>` - Remove authorization
   - Query param: institution_id (required)
   - Returns: { success: true }

**Catalog Requirements Endpoints:**

6. `GET /api/state-regulations/<state_code>/requirements` - List state requirements
   - Returns: { requirements: [...], count: N }

7. `GET /api/state-regulations/<state_code>/compliance` - Get compliance status
   - Query param: institution_id (required)
   - Returns: { compliance: [...], satisfied: N, partial: N, missing: N }

8. `PUT /api/state-regulations/<state_code>/compliance/<requirement_id>` - Update compliance
   - Query param: institution_id (required)
   - Body: { status, evidence_doc_id?, page_reference?, notes? }
   - Returns: { compliance: {...} }

**Program Approval Endpoints:**

9. `GET /api/state-regulations/<state_code>/programs` - List program approvals
   - Query param: institution_id (required)
   - Returns: { approvals: [...], count: N }

10. `POST /api/state-regulations/<state_code>/programs` - Add program approval
    - Body: { institution_id, program_id, board_name, approved?, approval_date?, expiration_date?, license_exam?, min_pass_rate?, current_pass_rate?, board_url?, notes? }
    - Returns: { approval: {...} }, 201

11. `PUT /api/state-regulations/<state_code>/programs/<approval_id>` - Update approval
    - Body: { approved?, approval_date?, expiration_date?, current_pass_rate?, notes? }
    - Returns: { approval: {...} }

12. `DELETE /api/state-regulations/<state_code>/programs/<approval_id>` - Delete approval
    - Returns: { success: true }

**Summary and Scoring Endpoints:**

13. `GET /api/state-regulations/<state_code>/readiness` - Get state readiness score
    - Query param: institution_id (required)
    - Returns: { readiness: { state_code, total, authorization_score, catalog_score, program_score, breakdown } }

14. `GET /api/state-regulations/summary` - Get all states summary
    - Query param: institution_id (required)
    - Returns: { states: [...], count: N }

15. `GET /api/state-regulations/renewals` - Get upcoming renewals
    - Query params: institution_id (required), days_ahead (default 90)
    - Returns: { renewals: [...], count: N }

**Preset Loading Endpoint:**

16. `POST /api/state-regulations/<state_code>/load-preset` - Load state requirements from preset
    - Reads data/state_requirements/{state_code}.json
    - Inserts requirements into state_catalog_requirements table
    - Returns: { loaded: N, state_code: "CA" }

All endpoints return proper error responses:
- 400 for missing required params
- 404 for not found resources
- 500 for server errors with error message
