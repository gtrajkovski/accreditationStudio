Create `static/js/keyboard-help.js` following the research pattern from 14-RESEARCH.md (lines 105-167):

**Class structure:**
```javascript
class KeyboardShortcutsModal {
  constructor() {
    this.modal = document.getElementById('keyboard-shortcuts-modal');
    this.backdrop = this.modal.querySelector('.modal-backdrop');
    this.content = this.modal.querySelector('.modal-content');
    this.focusableElements = 'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])';
    this.firstFocusableElement = null;
    this.lastFocusableElement = null;
    this.previousActiveElement = null;
  }

  open() { ... }
  close() { ... }
  handleKeydown(e) { ... }
}
```

**Key behaviors:**
1. **open()** method:
   - Store `this.previousActiveElement = document.activeElement` (for focus restoration)
   - Set modal display to 'flex'
   - Set `aria-hidden="false"`
   - Query focusable elements inside modal-content
   - Focus first focusable element
   - Add keydown event listener

2. **close()** method:
   - Set modal display to 'none'
   - Set `aria-hidden="true"`
   - Remove keydown event listener
   - Restore focus to `this.previousActiveElement`

3. **handleKeydown(e)** - Focus trap:
   - ESC key: call close()
   - Tab key forward: if on last element, wrap to first
   - Tab+Shift key backward: if on first element, wrap to last

**Global initialization:**
```javascript
window.KeyboardShortcuts = new KeyboardShortcutsModal();

// Listen for ? key globally (but not in input fields)
document.addEventListener('keydown', (e) => {
  if (e.key === '?' && !['INPUT', 'TEXTAREA'].includes(e.target.tagName)) {
    e.preventDefault();
    window.KeyboardShortcuts.open();
  }
});
```

**Important:** Follow accessibility best practices from research (lines 105-167). This is a WCAG 2.1 Level AA pattern.
