# v2.0 Requirements — Power User Features

## PWR-01: Bulk Remediation Wizard

**Goal:** One-click fix for all findings in scope

### Entry Points

- Dashboard: "Fix All Critical Findings" button
- Work Queue: "Fix Everything" toolbar action
- Document list: "Remediate All" batch action

## PWR-02: Packet Studio Wizard

**Goal:** Visual 5-step wizard from selection to export

### Step 1: Submission Type

```
┌──────────────────────────────────────────────┐
│  Step 1: What are you preparing?             │
├──────────────────────────────────────────────┤
│                                              │
│  ┌─────────────────┐  ┌─────────────────┐   │
│  │  📄 Self-Study  │  │  📝 Response    │   │
│  │  Full SER       │  │  Team report    │   │
│  └─────────────────┘  └─────────────────┘   │
│                                              │
│  ┌─────────────────┐  ┌─────────────────┐   │
│  │  📋 Annual      │  │  🏫 Teach-Out   │   │
│  │  Report         │  │  Plan           │   │
│  └─────────────────┘  └─────────────────┘   │
│                                              │
│                            [Next: Standards] │
└──────────────────────────────────────────────┘
```

### Step 2: Standards Selection

- Tree view of accreditor standards
- Checkbox per standard/sub-standard
- "Select All Section VII" shortcuts
- Previously addressed standards highlighted

### Step 3: Evidence Mapping

- Drag-drop evidence to standards
- Coverage indicator per standard (green/yellow/red)
- "Find Evidence" AI button per standard
- Overall coverage percentage

### Step 4: Narrative Generation

- AI-generated narrative per section
- Edit inline or regenerate
- Tone selector (formal/conversational)
- Citation style (inline/endnotes)

### Step 5: Preview & Export

- Full rendered preview
- Export format selection (DOCX, PDF, ZIP)
- Coverage validation gate (can't export if missing)
- Override with checkpoint if needed

### Acceptance Criteria

- [ ] 5-step wizard navigable via back/next
- [ ] Step state persists across browser refresh
- [ ] Evidence mapping shows coverage percentage
- [ ] Export blocked until coverage validated
- [ ] DOCX export produces valid document

## PWR-06: Document Workbench IDE

**Goal:** IDE-like split view with inline findings

### Layout

```
┌────────────────────────────────────────────────────────────────┐
│  Document Workbench                      [Program ▾] [Search] │
├──────────────┬───────────────────────────┬────────────────────┤
│ Documents    │  Document View            │ Findings           │
├──────────────┼───────────────────────────┼────────────────────┤
│ 📄 Catalog   │  Page 3 of 47             │ 🔴 Non-Compliant   │
│   ⚠️ 12 find │  ┌─────────────────────┐  │                    │
│ 📄 Enroll... │  │ The refund policy   │  │ Finding F-2847    │
│   🔴 3 crit  │  │ states students may │  │ Section VII.A.4   │
│ 📄 Handbook  │  │ [receive a full]    │←─│ "refund policy    │
│   ✅ OK      │  │ refund within 3     │  │ lacks required    │
│ 📄 Syllabus  │  │ days of enrollment. │  │ timeframe"        │
│   ⚠️ 1 find  │  │                     │  │                    │
│              │  │ [Highlighted text]  │  │ Confidence: 0.87  │
│              │  └─────────────────────┘  │                    │
│              │                           │ [Fix] [Dismiss]    │
│              │  [◀ Prev] Page 3 [Next ▶] │                    │
├──────────────┴───────────────────────────┴────────────────────┤
│ [View: Original ○ Remediated ○ Diff]     [Approve Changes]   │
└────────────────────────────────────────────────────────────────┘
```

### Features

- **Left panel:** Document list with finding counts and severity badges
- **Center panel:** Document content with page navigation
  - Inline highlighting for findings
  - Click highlight → shows finding in right panel
- **Right panel:** Findings tab
  - Selected finding details
  - Fix button → calls remediation agent
  - Dismiss with reason

### View Modes

- **Original:** Raw document as uploaded
- **Remediated:** AI-fixed version (if exists)
- **Diff:** Side-by-side comparison

### Acceptance Criteria

- [ ] Three-panel layout renders correctly
- [ ] Clicking finding in right panel scrolls to location in center
- [ ] Fix button triggers remediation and refreshes view
- [ ] Diff mode shows changes inline
- [ ] View mode persists across document changes
