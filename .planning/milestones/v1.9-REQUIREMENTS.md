# v1.9 Requirements — Regulatory Intelligence

## REG-09: Accreditor Package System

**Goal:** Repeatable onboarding structure per accreditor

### Directory Structure

```
src/accreditors/
├── __init__.py
├── registry.py          # Package discovery and access
├── base_package.py      # Abstract base class
├── accsc/
│   ├── __init__.py
│   ├── manifest.json    # { id, name, type, scope, standards_url }
│   ├── sources.py       # Official URLs and fetch cadence
│   ├── parser.py        # ACCSC-specific standards parser
│   └── mappings.py      # Crosswalk seeds (federal/state)
├── sacscoc/
│   └── ... (same structure)
├── hlc/
├── abhes/
└── coe/
```

### Base Package Interface

```python
class AccreditorPackage(ABC):
    @property
    def manifest(self) -> Dict: ...

    def get_sources(self) -> List[StandardsSource]: ...

    def parse_standards(self, raw_path: str) -> StandardsTree: ...

    def get_crosswalk_seeds(self) -> Optional[Dict]: ...

    def get_fetch_cadence(self) -> str: ...  # "daily", "weekly", "monthly"
```

### manifest.json Schema

```json
{
  "id": "ACCSC",
  "name": "Accrediting Commission of Career Schools and Colleges",
  "type": "institutional",
  "scope": "national",
  "standards_url": "https://www.accsc.org/Standards-and-Guidelines/...",
  "established": 1965,
  "recognized_by": ["USDE"],
  "fetch_cadence": "monthly"
}
```

### API

- GET /api/accreditors → list available packages
- GET /api/accreditors/{code} → package details
- POST /api/accreditors/{code}/fetch → trigger standards fetch

### Acceptance Criteria

- [ ] Base package interface defined
- [ ] Registry auto-discovers packages from src/accreditors/*/manifest.json
- [ ] ACCSC package fully functional (sources, parser, mappings)
- [ ] Other 4 packages have manifest + stubs
- [ ] API endpoints work

## REG-06: Federal Regulations Library

**Goal:** Structured bundles with applicability rules

### Federal Bundles

| Bundle | Applicability | Key Citations |
|--------|---------------|---------------|
| Title IV | title_iv_eligible = true | 34 CFR 668 |
| FERPA | has_students = true | 34 CFR 99 |
| Title IX | receives_federal_funds = true | 34 CFR 106 |
| Clery Act | title_iv_eligible = true | 34 CFR 668.46 |
| ADA/504 | all institutions | 28 CFR 35, 34 CFR 104 |
| Gainful Employment | for-profit OR certificates | 34 CFR 668.402 |
| COPPA | serves_minors = true | 16 CFR 312 |

### Bundle Schema

```python
@dataclass
class FederalBundle:
    id: str
    name: str
    short_name: str
    citations: List[str]
    applicability_rule: str  # Python expression against institution profile
    requirements: List[FederalRequirement]
    last_updated: str
    effective_date: str

@dataclass
class FederalRequirement:
    id: str
    citation: str
    title: str
    description: str
    evidence_types: List[str]
    common_violations: List[str]
    penalty_range: str
```

### Storage

```
src/regulatory/federal/
├── __init__.py
├── bundles.py           # Bundle loading and applicability
├── title_iv.json
├── ferpa.json
├── title_ix.json
├── clery.json
├── ada.json
├── gainful_employment.json
└── coppa.json
```

### API

- GET /api/federal/bundles → list all bundles
- GET /api/federal/bundles/{id} → bundle details
- GET /api/federal/applicable/{institution_id} → bundles applicable to institution
- GET /api/federal/search?q=... → search across all bundles

### Acceptance Criteria

- [ ] 7 federal bundles with citations and requirements
- [ ] Applicability rules evaluate against institution profile
- [ ] API returns applicable bundles for institution
- [ ] Search works across all bundles
