# v2.1 Requirements — Commercial Readiness

**Milestone:** v2.1
**Goal:** Make AccreditAI sellable to first paying customers (ACCSC career schools and accreditation consultants)

## Strategic Context

From competitive analysis (2026-03-29):
- AccreditAI is the **only AI-native product** in the accreditation management market
- Competitors (Watermark, SPOL, Weave) provide workflow management, not document-level AI
- **Blockers to first sale:** No multi-user, no onboarding, no cloud (cloud deferred to v2.2)

## Requirements

### R1: User Authentication
**Priority:** P0 (blocker)
**Phase:** 41

| Requirement | Acceptance Criteria |
|-------------|---------------------|
| R1.1 User registration | Email + password registration with validation |
| R1.2 User login | JWT session tokens with 24hr expiry |
| R1.3 Password reset | Token-based reset flow with 1hr expiry |
| R1.4 Session management | Validate sessions, logout invalidates token |
| R1.5 AUTH_ENABLED toggle | App works without auth when disabled (dev mode) |
| R1.6 First-run setup | Redirect to registration when no users exist |

### R2: Role-Based Access Control
**Priority:** P0 (blocker)
**Phase:** 42

| Requirement | Acceptance Criteria |
|-------------|---------------------|
| R2.1 Role hierarchy | owner > admin > compliance_officer > department_head > viewer |
| R2.2 Permission matrix | Each role has defined actions (see permission table) |
| R2.3 Institution scoping | Users only see their institution's data |
| R2.4 User invitation | Admins can invite users via email with role assignment |
| R2.5 Role management | Admins can change user roles |
| R2.6 User management UI | Page for listing, inviting, and managing users |

## Non-Functional Requirements

| Requirement | Criteria |
|-------------|----------|
| NFR1: Backward compatibility | AUTH_ENABLED=False preserves v2.0 behavior |
| NFR2: Performance | No regression in page load times |
| NFR3: Security | Passwords hashed with bcrypt, no PII in logs |
| NFR4: Testing | 90%+ test coverage for new code |

## Out of Scope for v2.1

- Cloud deployment (v2.2)
- Collaborative writing (v2.2)
- PostgreSQL migration (v2.2)
- SSO/SAML (v2.3)
- Public API (v2.3)
