1. PROJECT OVERVIEW
================================================================================

AccreditAI is an AI-powered platform for managing the ENTIRE accreditation
lifecycle of post-secondary educational institutions:

  - Self-evaluation
  - Document preparation
  - Exhibit collection
  - On-site visit readiness
  - Post-visit response

It audits against the FULL REGULATORY STACK:
  - Accreditor standards (ACCSC, SACSCOC, HLC, ABHES, COE)
  - Federal regulations (Title IV, DOE requirements)
  - State requirements
  - Professional licensure expectations

Deployment Model: Standalone Flask app, single-user localhost tool
Code Base: 121,194+ lines of code across Python, JavaScript, HTML/CSS

5. 24-AGENT TIER SYSTEM
================================================================================

Agents are organized into 7 tiers:

TIER 0 - Runtime & Governance:
  - ORCHESTRATOR: Workflow coordination and task dispatch
  - POLICY_SAFETY: Safety and policy enforcement
  - EVIDENCE_GUARDIAN: Evidence integrity protection

TIER 1 - Intake & Retrieval:
  - DOCUMENT_INTAKE: Document upload, parsing, indexing
  - PARSING_STRUCTURE: Section detection, structuring
  - PII_REDACTION: PII detection and redaction
  - RETRIEVAL_TUNING: Search optimization

TIER 2 - Standards & Regulatory:
  - STANDARDS_CURATOR: Standards library management
  - REGULATORY_STACK: Federal/state regulation tracking
  - STANDARDS_TRANSLATOR: Plain English explanations

TIER 3 - Compliance & Quality:
  - COMPLIANCE_AUDIT: Multi-pass compliance auditing (5 passes)
  - CONSISTENCY: Cross-document contradiction detection
  - RISK_SCORER: Risk prioritization, readiness scores
  - GAP_FINDER: Evidence gap identification

TIER 4 - Remediation & Authoring:
  - REMEDIATION: Generate redlines and finals
  - POLICY_AUTHOR: Policy document generation
  - EXHIBIT_BUILDER: Exhibit preparation
  - CHANGE_IMPACT: Change propagation analysis

TIER 5 - Submission & Defense:
  - NARRATIVE: Write issue narratives
  - CROSSWALK: Build compliance crosswalks
  - PACKET: Create submission packages
  - SITE_VISIT_COACH: Interview preparation

TIER 6 - Product Experience:
  - WORKFLOW_COACH: User guidance
  - LOCALIZATION_QA: Translation quality

Each agent:
  - Has defined tools (5-12 tools per agent)
  - Returns confidence scores (0-1)
  - Cites evidence for every claim
  - Triggers human checkpoints when confidence < 0.7

8. USER INTERACTION WORKFLOWS
================================================================================

IDEAL USER WORKFLOW:
--------------------
1. Create Institution
       |
2. Upload Documents (catalog, agreement, policies)
       |
3. Run Full Compliance Audit [One Click]
       |
4. Review Findings (Critical -> Advisory)
       |
5. Click "Fix It" -> Approve Redlines
       |
6. Re-run Audit -> Score Improves
       |
7. Generate Submission Packet
       |
8. Site Visit Prep with Evidence Explorer

STANDARD WORKFLOWS:
-------------------
1. Audit Enrollment Agreement (ACCSC):
   Ingestion -> Standards Librarian -> Evidence Mapper ->
   Compliance Audit -> Remediation -> Packet Assembly

2. Self-Study Preparation:
   Standards Librarian -> Evidence Mapper -> Crosswalk Builder ->
   Compliance Audit -> Narrative -> Packet Assembly

3. Site Visit Prep:
   Evidence Mapper -> Compliance Audit -> Risk Scorer ->
   Site Visit Prep

4. Substantive Change Filing:
   Substantive Change -> Evidence Mapper -> Narrative ->
   Packet Assembly

5. Annual Compliance Check:
   Ingestion (new docs) -> Policy Consistency ->
   Compliance Audit -> Risk Scorer -> Calendar Update

BULK REMEDIATION WIZARD FLOW:
-----------------------------
1. Select scope (all documents, by type, by program, by severity)
2. Preview affected documents and finding counts
3. Start bulk processing with SSE progress streaming
4. Review generated remediations with batch approval
5. Apply approved fixes, reject others

PACKET STUDIO WIZARD FLOW:
--------------------------
Step 1: Select submission type (self-study, response, teach-out, annual)
Step 2: Select accreditor and standards to address
Step 3: Map evidence to standards (drag-drop with coverage validation)
Step 4: Generate/review narratives (AI-assisted)
Step 5: Preview + Export (with blocking issue validation)

DOCUMENT WORKBENCH IDE:
-----------------------
- Left panel (20%): Document list with severity badges
- Center panel (50%): Document viewer with page navigation
  - Inline highlights for findings
  - View modes: Original, Remediated, Side-by-side Diff
- Right panel (30%): Findings list with details and actions
  - Click finding -> scrolls to position in document
  - Fix button -> preview modal -> apply remediation

10. QUALITY & SAFETY GUARDRAILS
================================================================================

AI SAFETY RULES:
----------------
1. NEVER fabricate evidence, policy text, or compliance claims
2. ALWAYS cite sources for every claim
3. ALWAYS label uncertainty (confidence < 0.7 requires human review)
4. REQUIRE human confirmation for compliance determinations
5. NEVER store student PII in embeddings or AI context
6. ALWAYS preserve original documents (originals/ folder is read-only)

CONFIDENCE THRESHOLD:
---------------------
- Agents check confidence against AGENT_CONFIDENCE_THRESHOLD (default 0.7)
- Below threshold triggers HumanCheckpoint for review
- All decisions logged in audit trail

EVIDENCE REQUIREMENTS:
----------------------
No claim accepted without:
- Confidence score (0-1)
- Evidence pointers (document_id, chunk_id, page)
- Standard citations

HUMAN CHECKPOINTS:
------------------
Triggered by:
- Low confidence (< 0.7)
- Legal/compliance determinations
- Final submissions
- Truth index changes
